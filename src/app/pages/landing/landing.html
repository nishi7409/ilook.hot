Why this exists

Built because nothing else quite fit. (and mostly vibe coded.)

Self-hostable fitness apps do exist — wger is probably the most well-known — but I couldn't find one with a UI I actually wanted to use every day. Good enough to work isn't the same as good enough to stick with. So I built what I wanted.

I'm a software engineer by trade, and the vast majority of this app was built with Claude Code. I still designed the entire architecture — data models, scheduling logic, component structure, tech stack — I just didn't feel like typing it all out after a full day of work. No shame.

ilook.hot runs on personal infrastructure. Expect occasional downtime, no SLA, and vibes-based incident response. If that bothers you, self-hosting is always an option.
